# NYC Traffic Safety — EDA & Interactive Dashboard

A comprehensive analysis of motor vehicle collisions in New York City (2023–2025), including geospatial data cleaning, exploratory data analysis, and an interactive Streamlit choropleth map.

---

## Project Structure

```
├── Crashes_EDA.ipynb          # Main analysis notebook (cleaning + EDA)
├── Map.py                     # Streamlit dashboard (interactive choropleth map)
├── monthly_crashes_gdf.geojson  # Generated by the notebook — required by Map.py
├── Crashes_20251231.csv       # Raw crash records (source: NYC Open Data)
├── BoroughBoundaries.csv      # Borough polygon boundaries (WKT format)
├── ModifiedZipCode.csv        # Modified ZIP code (MODZCTA) boundaries (WKT format)
└── README.md
```

## Dataset

| Property | Detail |
|---|---|
| **Source** | [Motor Vehicle Collisions – Crashes (NYC Open Data)](https://data.cityofnewyork.us/Public-Safety/Motor-Vehicle-Collisions-Crashes/h9gi-nx95/about_data) |
| **Size** | 270,000+ records, 29 columns |
| **Time span** | January 2023 – December 2025 |
| **Language** | Python 3.13 |

Each row is one police-reported crash event. Reports are required when someone is injured or killed, or when property damage exceeds $1,000.

## Notebook Overview (`Crashes_EDA.ipynb`)

### 1. Imports & Data Loading
Loads the raw crash CSV alongside borough and ZIP code boundary files.

### 2. Initial Data Inspection
Checks shape, column types, and duplicate records.

### 3. Data Cleaning
- **Column selection** — filters the 10 columns relevant to the analysis.
- **Spatial imputation** — fills missing `BOROUGH` and `ZIP CODE` values using GPS coordinates and a spatial join against official boundary polygons.
- **Feature engineering** — derives `YEAR`, `YEAR_MONTH`, `MONTH`, `HOUR`, and `WEEKDAY` from the crash timestamp.

### 4. Interactive Map Data Prep
Aggregates crashes, injuries, and deaths per ZIP code per month, joins the result to polygon geometries, and exports `monthly_crashes_gdf.geojson` for use by the Streamlit app.

### 5. Exploratory Data Analysis

**Time-series findings:**
- Annual crash counts declined steadily from 2023 through 2025.
- Monthly volumes stayed between ~6,000–8,000; January and February were consistently the safest months.
- Friday was the most crash-prone day of the week; crash volume peaked between 3 PM and 5 PM daily, with a secondary spike at midnight.

**Spatial findings:**
- Brooklyn and Queens had the highest total crash volumes (70,000+ incidents each).
- Brooklyn and the Bronx trended downward throughout the study period; Queens and Manhattan rebounded in 2025.
- Staten Island had the lowest volume overall but the highest crash fatality rate.
- ZIP code 10314 (Staten Island) ranked in the city-wide top 10 for total crashes — an outlier for its borough.

**Casualty findings:**
- 43% of crashes resulted in injuries; only 0.27% were fatal.
- Despite high raw casualty counts, Brooklyn and Queens had relatively low fatality rates.
- 2025 was the safest year, with fatality rates converging near 0.40% across all boroughs.

## Interactive Dashboard (`Map.py`)

A Streamlit app that lets users explore monthly crash density across NYC ZIP codes.

### Features
- **Year / Month dropdowns** — cascading selectors that show only valid month options for the chosen year.
- **Choropleth map** — ZIP code polygons shaded by monthly crash count (Yellow → Orange → Red).
- **Hover tooltip** — shows ZIP code, monthly crashes, injuries, and deaths on mouse-over.

### Prerequisites
The app reads `monthly_crashes_gdf.geojson`, which is produced by running `Crashes_EDA.ipynb` from start to finish.

## Usage

### Run the notebook
Open `Crashes_EDA.ipynb` in JupyterLab or VS Code and run all cells top-to-bottom.  
This will produce `monthly_crashes_gdf.geojson` in the project directory.

### Launch the dashboard
```bash
streamlit run Map.py
```
Then open the URL shown in the terminal.
